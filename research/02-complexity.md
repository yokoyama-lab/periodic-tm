# 課題2: 二対合分解の計算量 — 上界と P 保存（証明済み）、下界（予想＋障害の記録）

## 定理 2.1（クエリ上界）— 証明済み

**定理.** f をオラクル、fⁿ = id とする。軌道反転による分解 f = ι₁ ∘ ι₂ は、
ι の 1 評価あたり f への高々 ℓ ≤ n 回のクエリで計算できる
（ℓ は入力の軌道長）。

**証明.** ι(x) の計算は x の軌道走査（x, f x, …, x に戻るまで: ちょうど
ℓ クエリ）で軌道全体をメモリに得る。基点（軌道の最小元）・x の位置 k・
反転先はすべてこのリストの読み出しであり追加クエリ不要。∎

`themes/02` の bench.py の実測 2n/評価は、実装が基点探索と反転で走査を
2 回行うためで、1 回に統合すれば n に落ちる（定数因子の差のみ）。

## 定理 2.2（P 保存、位数固定）— 証明済み

**定理.** n を固定した定数とする。f が多項式時間計算可能な全単射で
fⁿ = id なら、ι₁, ι₂ も多項式時間計算可能にとれる。

**証明.** 多項式時間関数の像のサイズは多項式で膨らむが、合成回数は
高々 n（定数）なので、f^k(x) (k ≤ n) はすべて多項式時間・多項式サイズ。
軌道は高々 n 元なので、辞書式最小の基点選択・位置決定・反転はすべて
多項式時間。∎

**注意（n を入力にすると壊れる）.** n を二進で入力に含めると軌道長が
入力長の指数になり、この構成は多項式時間でなくなる。「n 可変で P を
保存する分解はあるか」は開いた問題として残す（おそらく否定的:
軌道走査を避ける手段が見当たらない。下の下界予想と同根）。

## 予想 2.3（オラクル下界）— 未解決、障害を記録

**予想.** 任意のオラクル機械の組 (M₁, M₂) が全ての位数 n の全単射 f に
対して有効な分解 (ι₁, ι₂) = (M₁^f, M₂^f) を与えるなら、最悪ケースで
1 評価あたり Ω(n) クエリが必要。

### 敵対的引数の試みと、それを阻む2つの障害

**素朴な議論.** ι₂(x) は x の軌道内の位置に依存する。o(n) クエリでは
軌道の全貌が見えないので、敵対者は未クエリ部分の閉じ方を変えて
出力を偽にできる — に見える。しかし:

**障害A（軌道間ペアリングの自由度）.** ι f ι = f^{-1} は ι が軌道を
「向きを逆にして」軌道へ写すことしか強制しない。ι は x の軌道を
**別の軌道**へ写してもよい。敵対者が x の軌道の閉じ方を変えても、
アルゴリズムは相方の軌道の選び方でつじつまを合わせる余地がある。
下界証明は「どの相方を選んでも矛盾する」ことを示す必要があり、
これは単一入力の議論では閉じない（ι の大域的整合性、すなわち
別入力での実行との整合を使う必要がある）。

**障害B（対の共謀）.** ι₁ と ι₂ は独立に正しい必要はなく、積が f で
ありさえすればよい。ι₂ が軌道構造を「誤解」しても ι₁ が同じ誤解を
すれば積は正しくなりうる。ただし両者とも**対合**でなければならず、
対合性が誤解の自由度を大きく削る — ここが攻め口だと考えている。

### 部分的な足がかり（証明の設計図）

制限クラスでの下界なら落とせる見込みが高い:
「**軌道保存的**な分解（ι が各軌道を自分自身に写す）は、1 評価あたり
最悪 ⌈(ℓ−1)/2⌉ クエリを要する」。このときは障害Aが消え、軌道内の
反転中心の位置が情報理論的に ℓ/2 ビット級の不確定性を持つため、
標準的な敵対者論法が機能するはず。まずこれを厳密化し、
一般ケースは「軌道保存的に正規化できるか」（できれば下界が移る、
できない反例があればそれ自体が面白い）に帰着させるのが次の一手。
