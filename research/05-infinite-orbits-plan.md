# 攻略計画: 無限軌道上の位置・所属の決定不能性

00-plan.md の結論「全課題の障害は無限軌道の位置・所属に集約される」を
受け、この一点を正面から解くための計画。目標は次の主定理である。

> **主定理（目標）.** 計算可能な全単射 f : N → N で、計算可能な
> 反転対称性（ι f ι = f^{-1} なる計算可能な対合 ι）を持たないものが
> 存在する。さらに「持つ ⟺ 何か」の完全な特徴づけを与える。

## Phase 0: 問題の分解と正しい語彙（1〜2週間）

無限軌道の困難を3つの決定問題に分離する。

- **P1 所属**: x, y が同じ軌道か（∃k ∈ Z. f^k(x) = y）。Σ⁰₁。
- **P2 位置**: 同じ軌道のとき、符号つき距離 k の計算。
  補題 4.3 より鎖が1本なら計算可能。困難は P1 との合成で生じる。
- **P3 正準中心**: 各軌道からの一様な代表元選択。
  軌道は c.e. 集合なので min は極限計算可能どまり。

**語彙の選定が最初の作業**: f の軌道同値関係 E_f は c.e. 同値関係
（ceer）である。ceer の理論（Andrews–Sorbi 系列の還元・完全性理論）は
まさに P1 の難しさを測る道具を既に持っている。反転対称性 ι は
E_f の自己同型を誘導するから、「ι の非存在」は「ある ceer の
計算可能自己同型の非存在」に翻訳できるはずである。
**タスク 0.1**: ceer 文献の調査（Andrews–Sorbi のサーベイ、
computable reducibility of equivalence relations）。既に解かれていないか
（軌道同値関係の自己同型問題として）を最初に確認する — ここで
既存定理が見つかれば Phase 2–3 は引用で済む。/related-work で実施。

## Phase 1: 実現定理（易しい足場、1週間）

**命題（見込み: 容易）.** 任意の Σ⁰₁ 集合 S に対し、計算可能な全単射 f で
「軌道所属問題が S と Turing 同値」なものが存在する。
構成: e ∈ S の列挙時刻 s に応じて2本の鎖を連結する標準的コーディング。
これで「無限軌道の所属はいくらでも難しくできる」を確定させ、
主定理の構成の部品（鎖の連結操作のライブラリ）を整備する。
**成果物**: 補題と、鎖連結を Python でシミュレートする
`chain_builder.py`（Phase 2 の検証基盤を兼ねる）。

## Phase 2: 単一要件の撃破（核心その1、2〜4週間）

04 ノートの設計図の最小単位を完成させる。

**補題（目標）.** 任意の部分計算可能関数 φ に対し、計算可能な全単射
f_φ で「φ は f_φ の反転対称性ではない」ものを一様に構成できる。

戦略: 鎖片（有限パス）を並べ、φ が入力 u で収束して v = φ(u) を
返したら、ι f ι = f^{-1} が強制する「u と v の距離の写し取り」を
偽にする向きに鎖片を連結する。φ が収束しなければ既定の連結で
f_φ を全域にする（**期限管理**: 段階 s までに番地 < s の f 値を
必ず確定させる規約で全域性を保証。04 ノートの技術的問題 1 の解答案）。

**検証**: 構成を `chain_builder.py` 上で実装し、具体的な φ
（例: 「常に恒等」「軌道走査で反転を試みる正直な戦略」）に対して
実際に確約が偽になることを機械検査する。
**判定点**: この補題が書き下せなければ主定理は現構成では無理。
その場合は Phase 0 の ceer 翻訳経由の間接証明へ切り替える。

## Phase 3: 優先度法への持ち上げ(核心その2、1〜2ヶ月）

全要件 R_e（φ_e は反転対称性でない）を同時に満たす。
04 ノートの技術的問題 2（別鎖ペアリングでの逃げ）と 3（損傷管理）が
ここで現れる。

- **問題2への案**: 鎖に計算可能な「長さ勾配マーカー」（鎖片内の
  有限ブロック構造）を刻み、ι が鎖を別の鎖に写すには構造が一致する
  相手が必要になるようにする。要件ごとに構造を唯一化すれば
  自己反転を強制でき、パリティ論法が不要になる（04 の案より筋がよい）。
- **問題3への案**: R_e の確約破壊は自分専用の鎖群の再連結だけで行う
  （鎖の要件別割当て）。これなら要件間の干渉が消え、**損傷なし
  （injury-free）構成**にできる見込みすらある。まずこれを試し、
  ダメなら有限損傷へ。

**成果物**: 主定理前半（非存在例）の完全な証明。
**リスク**: ι が「無限個の鎖にまたがる撹拌」で逃げる筋を
潰し切れるか。逃げ筋が見つかったら、それ自体を補題化する
（「反転対称性は軌道集合の対合的置換を誘導する」は代数的に自明
なので、撹拌は鎖単位の置換に限られる — この整理を証明の冒頭に置く）。

## Phase 4: 特徴づけ（主定理後半、2〜4週間）

非存在例が取れたら境界線を引く。候補となる同値条件（強い順）:

  (a) f が計算可能な対合的反転を持つ（求めるもの）
  (b) f が計算可能な反転を持つ（ι の対合性を落とす: ∃g 計算可能,
      g f g^{-1} = f^{-1}）
  (c) 軌道ごとの中心選択関数が計算可能
  (d) P1（所属）+ P3（中心）が f から一様に計算可能

**問い 4.1**: (b) ⇒ (a) か。GKM は群論的設定で「反転可能なら対合的
反転がとれる」ことを使う（g f g⁻¹ = f⁻¹ なら g f も反転で
(gf)² = …）。この変換 g ↦ 適切な補正 が計算可能性を保つかを検査する。
保つなら (a) = (b) で特徴づけは「計算可能反転可能性」に一本化される。
**問い 4.2**: (a) ⟺ (d) の予想を証明する。⇐ は定理 4.1 の構成の
一般化で見込みあり。⇒ が非自明（ι から中心を復元できるか。
ι の不動点/2周期点が中心候補を与える — 各鎖の反転は不動点を
0 or 1 個持つことに注意。鎖反転に不動点がない場合(半整数中心)の
扱いが鍵）。

## Phase 5: 他課題への還流（継続）

- **課題2の下界**: Phase 2 の敵対的連結は、そのままオラクル下界の
  敵対者である（有限版に焼き直すだけ）。障害A（軌道間ペアリング）への
  解答も Phase 3 の勾配マーカーが流用できる見込み。
- **課題1の不可能性**: 主定理が取れれば「構文クラスの決定可能な
  列挙 + 位置決定の不能性」の衝突として不可能性定理の部品になる。

```
                 └── 既存定理が見つかれば Phase 2–3 をスキップして Phase 4 へ
```
