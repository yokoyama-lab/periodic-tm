# 課題4: 計算可能な時間対称性 — 局所有限定理（証明済み）と反例の設計図

## 定理 4.1（局所有限分解定理）— 証明済み

**定理.** f : N → N を計算可能な全単射とし、すべての軌道が有限
（各 x にある ℓ ≥ 1 で f^ℓ(x) = x）とする。このとき計算可能な対合
ι₁, ι₂ が存在して f = ι₁ ∘ ι₂。

これは本リポジトリの有限位数定理（fⁿ = id）の真の一般化である。
fⁿ = id は「軌道長が一様に n で抑えられる」場合であり、本定理は
一様な上界を要求しない（軌道長は非有界でよい）。

**証明.** x が与えられたとき、x, f(x), f²(x), … を x に戻るまで列挙する。
軌道は有限なので、この走査は必ず停止し（停止時刻の事前の計算可能な
上界は不要である。停止性は各入力ごとの有限性から従う）、軌道
O = (b, f(b), …, f^{ℓ-1}(b)) と長さ ℓ を得る。基点 b は O の数値最小元
という正準選択をとる（走査結果から計算可能）。x = f^k(b) として

  ι₂(f^k(b)) := f^{ℓ-k mod ℓ}(b),   ι₁(f^k(b)) := f^{(1-k) mod ℓ}(b)

と定める。基点が正準なので ι₁, ι₂ は well-defined な全域計算可能関数で、
軌道ごとの反転だから ι₂² = ι₁² = id、そして
ι₁(ι₂(f^k b)) = ι₁(f^{-k} b) = f^{1+k}(b) = f(f^k b)。∎

数値実験による確認: `locally_finite.py`（軌道長が非有界の f で
ι² = id と f = ι₁∘ι₂ を大量サンプル検査、全て通過）。
**Lean 機械化済み**: `lean/FiniteOrderTM/LocallyFinite.lean`
（`locallyFinite_eq_two_involutions`。数学的核＝任意の点ごと周期的
写像の二対合分解。sorry ゼロ、`OrderDividing` からの橋渡し系付き）。

**系.** 計算可能な全単射で2対合の積に書けないものは、無限軌道を持つ。

## 命題 4.2（無限位数でも書けることがある）— 証明済み

f(x) = x + 1（Z 上）は無限位数だが、ι₂(x) = −x, ι₁(x) = 1 − x で
f = ι₁ ∘ ι₂（無限二面体群 D∞）。よって「有限軌道」は必要条件ではない。
境界は軌道の有限性ではなく、次の 4.3–4.5 の位置・所属の決定可能性にある。

## 補題 4.3（単一 Z 鎖は常に可逆化できる）— 証明済み

f の軌道がただ1本の両側無限鎖（Z のコピー）なら、計算可能な反転対称性を
持つ。**証明.** 基点 b を固定。任意の u に対し k = 0, +1, −1, +2, −2, … の
順に f^k(b) を計算すれば、鎖が1本なのでいつか u に一致し、位置
pos(u) = k が計算できる。ι(u) := f^{−pos(u)}(b) が計算可能な反転。∎

## 反例構成の設計図（未完 — ここが研究の最前線）

**目標.** 計算可能な全単射 f で、計算可能な ι（ι f ι = f^{-1}）を
一切持たないものを構成する。

補題 4.3 より鎖1本では作れない。**無限本の Z 鎖**が必要で、障害は
「u がどの鎖に属すか」「鎖ごとの反転中心をどう一様に選ぶか」にある。
鎖は c.e. 集合だが、その最小元（正準中心の自然な候補）は
極限計算可能でしかない — ここが定理 4.1 の証明が破れる正確な地点。

**構成方針（優先度法）.** 各候補 ι = φ_e に対し要件
R_e: 「φ_e は f の反転対称性ではない」を立てる。鎖 C_e を φ_e 専用に
用意し、φ_e が C_e のある元 u で収束して v = φ_e(u) を出力したら、
それは反転中心の位置への確約である（ι f ι = f^{-1} は中心から u までの
距離を v が写し取ることを強制する）。その確約後に C_e の f 値の
未定義部分を非対称に延長し、確約を偽にする。

**残る技術的問題（正直な記録）:**
1. f は全域計算可能でなければならないので、「延長を保留する」ことは
   できない。各段階で f の値を確定しつつ鎖の接続だけを保留する
   （有限鎖片を並べておき、後で連結順を選ぶ）必要がある。鎖片の連結は
   f の未定義点の値の選択として実装できるが、全域性のためには
   各点の値をいつかは確定させる期限管理が要る。
2. φ_e の確約が「u と v が同じ鎖に属す」ことを仮定しない場合
   （別鎖への対応で逃げる場合）、鎖の本数のパリティで詰ませる案:
   鎖を奇数本にし、少なくとも1本は自己反転を強いられるようにする。
   ただし「どの鎖が自己反転か」を敵対的に動かすには鎖の同一性を
   段階的に融合できる構成が要り、1 の期限管理と衝突しうる。
3. 有限損傷（finite injury）で足りるか。R_e の損傷は鎖片の再連結で
   起きるが、再連結は他の R_i の確約も壊しうる。優先順位で下位のみ
   壊す設計にできれば有限損傷で済む見込み。

**次の一手**: まず要件1本（単一の φ_e に勝つ）の構成を完全に書き下し、
Python で段階的構成をシミュレートして矛盾がないか機械検査する。
これが通れば優先度法への持ち上げは定型作業に近い。
