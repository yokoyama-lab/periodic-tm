# テーマ02: 二対合分解の計算量解析

## 問題設定

分解定理 f = ι₁ ∘ ι₂ は「計算可能」であることしか言っていない。
では ι₁, ι₂ は f と比べて**どれくらい遅くなるか**。

- ι の 1 回の評価に必要な f の呼び出し回数は？（軌道の折り返しには
  軌道の走査が要る → 素朴には O(n) 回）
- 時間計算量・領域計算量のクラスは保存されるか（f が P なら ι も P か）
- 下界はあるか（O(n) 回の f 呼び出しは避けられないか）

## 最初の一歩

```bash
python3 bench.py    # 分解のコストを位数 n ごとに実測
```

`bench.py` は `../../finite_order.py` の `decompose` を計装し、
ι の 1 評価あたりの f 呼び出し回数を位数 n を変えて数える。
まず実測でオーダーの仮説を立て、それを証明するのがこのテーマの流れ。

## マイルストーン
