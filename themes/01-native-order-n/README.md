# テーマ01: 位数 n をネイティブに実現する単一機械モデル（未解決問題）

## 問題設定

ITM は「機械の構文的対称性」だけで f² = id を保証する。同じことを位数 n で
やりたい。すなわち、次を満たす機械クラス C_n を設計せよ。

1. **健全性**: C_n の任意の機械が計算する関数 f は fⁿ = id を満たす。
2. **完全性**: fⁿ = id なる任意の計算可能な f は C_n のある機械で計算できる。
3. **構文性**: 所属判定（機械が C_n に入るか）が決定可能。

n = 2 は Nakano の ITM が解決済み。n ≥ 3 が未解決。

## なぜ素朴な方法が失敗するか（ここが出発点）

n = 2 の証明は「実行の逆再生 = 逆関数」という時間反転を使う。時間反転は
位数 2 の操作なので Z/2 とは噛み合うが、「1/3 だけ逆再生する」に対応する
機械操作は存在しない。よって状態集合に Z/n 対称性を入れるだけでは
健全性は出ても完全性が出ない。詳しくは本リポジトリの二対合分解定理
（`lean/FiniteOrderTM/Basic.lean`）と no-go の分析を参照。

## 最初の一歩

1. `finite_order.py` を動かし、位数 3 の関数が 2 つの対合に割れることを確認する。
2. 「ITM 2 台の直列合成 ι₁ ∘ ι₂」を1台の機械とみなす合成モデルを定義してみる
   （分解定理より完全性は自動で出る。問題は健全性と構文性）。
3. 対合でなく「巡回置換シフト」を持つ多テープ機械など、時間反転以外の
   位数 n 操作の候補を列挙して健全性を判定する。
